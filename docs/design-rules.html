AisleFinder Design Rules

The living style guide for the app. Source of truth: the CSS-variable block in src/AisleFinder.jsx (theme + shared classes) and src/components/Logo.jsx (logo art). If you change values there, update this page to match.

Typography src/AisleFinder.jsx — shell fontFamily; everything inherits

One family everywhere: Arial, 'Helvetica Neue', Helvetica, sans-serif. Components must use fontFamily: 'inherit', never their own stack. Arial has only two real weights — regular (400) and bold (700); don't use 350/600/800.

Core palette src/AisleFinder.jsx — :root + prefers-color-scheme block

The scheme is "Evergreen Chrome": a solid forest-green top bar in both modes over a calm, green-tinted body. The palette is deliberately tiny: six neutrals, one green pair, one chrome green, one amber, one error tone per scheme. No blues anywhere. Every other --af-* variable (highlight, error box, disabled, toast, celebrate, popup, chrome text…) is an alias or an alpha tint of these — never a new hex. Components reference variables only.

Rules

  • Icons: always Font Awesome (<i className="fa-solid fa-...">), never emojis. Kit loaded in public/index.html; prefer older icon names (fa-cog over fa-gear) if one doesn't render.
  • Logo: the shelf-"A" monogram, all green (no blue in the mark). Art lives in src/components/Logo.jsx and public/logo.svg — keep them identical.
  • Colors: component code references var(--af-*) only. The core set is 11 colors per scheme — before adding a hex, derive from an existing core color (alias or alpha tint) instead. No blues; no gradients outside the logo.
  • Chrome: the top bar sits on --af-chrome (forest green) in both modes. Controls inside it use the --af-chrome-* white-alpha tones, scoped via the .af-topbar class — never the body tones.
  • Font: set once on the app shell; components inherit. No per-component font stacks.
  • Buttons: solid green = the "go shopping" happy path and primary actions, ghost/outline = secondary. Hover lifts 1–2px with a soft shadow.
  • Radii: 6px small controls, 8–10px buttons/inputs, 12px cards, 18px sheets, 999px chips.
  • Screens over popups: full screens for destinations (list, lists, history, shop); bottom sheets for quick tasks (account, share, store); center popups only for tiny settings.
  • Dark mode: automatic via prefers-color-scheme — test both before shipping UI changes.
